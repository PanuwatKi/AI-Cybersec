## 1. ภาพรวมโปรเจกต์ (ปัญหา + ทางแก้)

**ปัญหา:** แก๊งคอลเซ็นเตอร์/มิจฉาชีพหลอกคนไทยสูญเงินปีละหลายหมื่นล้านบาท เหยื่อจำนวนมากคือ
ผู้สูงอายุที่แยกไม่ออกว่าสายไหนคือมิจฉาชีพ เพราะคนร้ายพูดน่าเชื่อถือและกดดันให้รีบโอน

**ทางแก้ของเรา:** อุปกรณ์ที่ "ฟังบทสนทนาของสายโทรเข้า" แล้วใช้ AI วิเคราะห์แบบเรียลไทม์ว่าน่าจะเป็น
มิจฉาชีพแค่ไหน แล้วเตือนด้วย **ไฟ 3 สี + เปอร์เซ็นต์ความมั่นใจ**:

| สี | ความหมาย | เกณฑ์ (% โอกาสเป็นมิจฉาชีพ) |
|----|----------|------------------------------|
| 🔴 แดง | เสี่ยงเป็นมิจฉาชีพสูงมาก | ≥ 70% (มีเสียง Buzzer เตือนด้วย) |
| 🟡 เหลือง | น่าสงสัย ควรระวัง | 40–69% |
| 🟢 เขียว | ปลอดภัย | < 40% |

**กลุ่มเป้าหมาย:** ผู้สูงอายุ และคนทั่วไปที่รับสายแปลกหน้า
**ธีมที่เลือก:** Health & Well‑Being (ยกระดับคุณภาพชีวิต ลดความเครียดและความสูญเสียจากภัยมิจฉาชีพ)

## 2. อุปกรณ์ที่ต้องใช้ทั้งหมด

### อุปกรณ์ที่ผู้จัดแจกให้ (ไม่ต้องเตรียม)
- **Arduino UNO Q (RAM 2GB)** — บอร์ดสมองกล รัน Linux + Python ได้
- **Modulino 7 โมดูล** — Buttons, Pixels, Knob, Thermo, Distance, Movement, Buzzer
- **กล้องเว็บแคม** — ใช้ "ไมโครโฟน" ของมันรับเสียงสาย
- ที่ชาร์จ GaN 65W, USB Hub, กล่องถ่ายรูปสินค้า

### อุปกรณ์ที่ทีมต้องเตรียมไปเอง
- **โน้ตบุ๊ก Core i5 ขึ้นไป** (ตัวรัน AI หลัก) — ลงโปรแกรมและทดสอบให้เสร็จก่อนไป
- **ปลั๊กต่อพ่วง**
- (แนะนำ) สาย USB‑C สำรอง + ไมโครโฟน/หูฟังเล็ก ๆ เผื่อไมค์เว็บแคมไม่ชัด

### โมดูลที่โปรเจกต์เราใช้จริง (ที่เหลือไม่ต้องใช้)
| อุปกรณ์ | หน้าที่ในงานเรา |
|---------|------------------|
| โน้ตบุ๊ก | รัน AI: แปลงเสียงเป็นข้อความ (Whisper) + วิเคราะห์มิจฉาชีพ (โมเดลของเรา) |

## 3. การทำงานของระบบโดยละเอียด

### ภาพรวมการไหลของข้อมูล (Data Flow)
```
🎤 เสียงสายโทรเข้า (ไมค์)
   → 🧠 Whisper  : แปลงเสียงพูดไทย → ข้อความ
   → 🔢 TF-IDF   : แปลงข้อความ → ตัวเลข (นับคำสำคัญ + คู่คำ + ชิ้นส่วนตัวอักษร)
   → 🤖 ComplementNB : คำนวณ % โอกาสเป็นมิจฉาชีพ
   → 🎨 เกณฑ์ 3 สี : แปลง % → แดง/เหลือง/เขียว
   → 🔌 ส่งผ่านสาย USB → Arduino UNO Q
   → 💡 Modulino Pixels ขึ้นไฟ + 🔊 Buzzer เตือนถ้าแดง
```

### อธิบายแต่ละส่วนแบบเข้าใจง่าย (ใช้ตอบกรรมการได้)
- **pythainlp (ตัวตัดคำ newmm):** ภาษาไทยเขียนติดกันไม่มีเว้นวรรค AI เลยต้อง "ตัดคำ" ก่อน
  เช่น `โอนเงินด่วน` → `โอน | เงิน | ด่วน`
- **TF-IDF (แปลงข้อความเป็นตัวเลข):** ให้ "น้ำหนัก" กับคำที่เจอบ่อยในข้อความมิจฉาชีพแต่ไม่ค่อยเจอ
  ในข้อความปกติ เช่น `โอนเงิน`, `กดลิงก์`, `OTP`, `อายัดบัญชี` จะมีน้ำหนักสูง
  - เรามองทั้ง **ระดับคำ** (และคู่คำ เช่น "โอน เงิน") และ **ระดับตัวอักษร** (ชิ้นละ 2–4 ตัว)
  - การมองระดับตัวอักษรทำให้ระบบ **ทนต่อการสะกดเพี้ยน** เช่นเวลาเสียงพูดถูกถอดผิดเป็น
    "ทนาคาน" หรือ "โอที่พี่" ก็ยังจับได้ (เราทดสอบแล้วได้ผลจริง)
- **ComplementNB (โมเดล Naive Bayes):** คำนวณความน่าจะเป็นว่าข้อความเป็นมิจฉาชีพ จากคำที่ปรากฏ
  จุดเด่นคือ **เร็วมาก ใช้ข้อมูลน้อยก็เรียนรู้ได้ และรันออฟไลน์บนเครื่องเล็กได้**
- **Whisper (แปลงเสียงเป็นข้อความ):** AI ของ OpenAI ที่ถอดเสียงพูดเป็นตัวอักษร รองรับภาษาไทย
  และ **ทำงานออฟไลน์** (ดาวน์โหลดโมเดลครั้งเดียว) — เหมาะกับหน้างานที่อาจไม่มีเน็ต

### วิธีติดตั้งและรัน (สำหรับลงในโน้ตบุ๊กที่จะใช้แข่ง)
```bash
# 1) ติดตั้งไลบรารีหลัก
pip install -r requirements.txt
# 2) ติดตั้งไลบรารีโหมดเสียง (Whisper)
pip install -r requirements-audio.txt
# 3) เทรนโมเดล (สร้าง scam_model.pkl + vectorizer.pkl)
python train_model.py        # พิมพ์ exit เพื่อออกจากโหมดทดสอบ
# 4) รันระบบสาธิต
python predict.py                 # โหมดพิมพ์ข้อความ
python predict.py --listen        # โหมดพูดใส่ไมค์
python predict.py --list-devices  # ดูรายชื่อไมโครโฟน
python predict.py --port COM3 --listen   # เสียง + ส่งไฟไป Arduino
```
> ⚠️ สำคัญ: รัน `--listen` ครั้งแรก **ตอนมีเน็ต** หนึ่งครั้ง เพื่อให้ Whisper ดาวน์โหลดโมเดลเก็บไว้
> หลังจากนั้นใช้งานออฟไลน์ได้ตลอด

### ไฟล์ในโปรเจกต์
| ไฟล์ | หน้าที่ |
|------|---------|
| `train_model.py` | เทรนโมเดล + วัดผล + บันทึก .pkl |
| `predict.py` | ระบบสาธิต (พิมพ์/เสียง/ต่อบอร์ด) |
| `arduino_unoq_modulino.ino` | โค้ดฝั่ง Arduino คุมไฟ Modulino |
| `Dataset1–4.csv` | ข้อมูลฝึก AI (เพิ่มไฟล์ใหม่ได้ ระบบโหลดอัตโนมัติ) |
| `scam_model.pkl`, `vectorizer.pkl` | โมเดลที่เทรนแล้ว |
| `requirements*.txt` | รายการไลบรารี |

## 4. จุดเด่นที่ควรพูดตอนนำเสนอ ✅

1. **แก้ปัญหาจริงที่รุนแรงระดับชาติ** — ภัยคอลเซ็นเตอร์ คนไทยสูญเงินมหาศาล เหยื่อหลักคือผู้สูงอายุ
2. **ทำงานออฟไลน์ 100%** — ไม่ต้องส่งเสียงขึ้นคลาวด์ ⇒ เป็นส่วนตัว ปลอดภัย และใช้ที่ไหนก็ได้
3. **เตือนเข้าใจง่ายด้วยไฟ 3 สี + %** — ผู้สูงอายุดูปุ๊บรู้ปั๊บ ไม่ต้องอ่านเยอะ
4. **ทนต่อการถอดเสียงผิดพลาด** — เพิ่มฟีเจอร์ระดับตัวอักษร ทำให้แม้ Whisper สะกดเพี้ยนก็ยังจับได้ (สาธิตให้ดูได้)
5. **AI เบา รันบนอุปกรณ์เล็ก/ราคาถูกได้** — ขยายผลไปติดตามบ้านผู้สูงอายุได้จริง
6. **วัดผลอย่างเป็นวิทยาศาสตร์** — Cross-Validation 5 รอบ ความแม่นยำ ~93–94% ครอบคลุมมิจฉาชีพ 10+ ประเภท
7. **ใช้ฮาร์ดแวร์ที่แจกอย่างมีความหมาย** — ไมค์เว็บแคมรับเสียง + Modulino แสดงผล
8. **ต่อยอดได้** — เพิ่มข้อมูลใหม่ได้ทันที (ระบบโหลด CSV อัตโนมัติ), เชื่อมกับสายโทรศัพท์/แอปได้ในอนาคต

| Naive Bayes สมมติว่าคำเป็นอิสระต่อกัน | ไม่ตรงความจริงเป๊ะ | "แลกกับความเร็ว ความเบา และอธิบายง่าย เหมาะกับอุปกรณ์เล็ก" |

| รองรับเฉพาะภาษาไทย | ยังไม่รองรับภาษาอื่น | "วางแผนต่อยอดหลายภาษาในอนาคต" |
| ไม่ใช่คำตัดสินสุดท้าย | เป็นผู้ช่วยเตือน ไม่ใช่แทนวิจารณญาณคน | "ออกแบบเป็นสัญญาณเตือนภัยล่วงหน้า ช่วยให้คนฉุกคิดก่อนโอน" |

## 7. คำถามที่กรรมการอาจถาม + แนวตอบ

- **"AI ทำงานยังไง?"** → ตัดคำไทย → แปลงเป็นตัวเลขด้วย TF-IDF (ให้น้ำหนักคำที่บ่งชี้มิจฉาชีพ) →
  Naive Bayes คำนวณ % → แปลงเป็น 3 สี
- **"ข้อมูลมาจากไหน / มีกี่แบบ?"** → ทีมรวบรวมและสร้างเอง ครอบคลุม 10+ ประเภท (พัสดุ, ตำรวจ/ปปง.,
  ธนาคาร, เงินกู้, รางวัล, ลงทุน, romance, แอบอ้างหน่วยงาน, ลิงก์ฟิชชิง, งานออนไลน์) + บทสนทนาปกติ
- **"แม่นแค่ไหน วัดยังไง?"** → Cross-Validation 5 รอบ ~93–94% (เป็นค่าที่น่าเชื่อถือกว่าวัดครั้งเดียว)
- **"ถ้าถอดเสียงผิดล่ะ?"** → เพิ่มฟีเจอร์ระดับตัวอักษร ทำให้ทนต่อการสะกดเพี้ยน + มีโหมดพิมพ์สำรอง
- **"ต่างจากแอปบล็อกสายทั่วไปยังไง?"** → แอปทั่วไปบล็อกตาม "เบอร์" ที่เคยมีคนรายงาน แต่ของเรา
  วิเคราะห์ "เนื้อหาบทสนทนา" จึงจับเบอร์ใหม่/สคริปต์ใหม่ได้ และทำงานออฟไลน์
- **"เอาไปใช้จริงยังไง?"** → ทำเป็นกล่องเล็กตั้งข้างโทรศัพท์บ้านผู้สูงอายุ หรือพัฒนาเป็นแอปในอนาคต
